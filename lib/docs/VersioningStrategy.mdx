  <div className="sb-section-item">
    <h3 className="sb-section-item-heading">PATCH (0.0.x)</h3>
    <p className="sb-section-item-paragraph">
      Patch versions include backward-compatible bug fixes or small improvements.
    </p>
    <p className="sb-section-item-paragraph">
      <b>Examples:</b>
    </p>
    <ul>
      <li>Fixing bugs</li>
      <li>Improving performance</li>
      <li>Updating documentation</li>
      <li>Refactoring internal code (with no API changes)</li>
      <li>Fixing accessibility issues</li>
      <li>Adjusting styles that don't impact component behavior</li>
    </ul>
    <p className="sb-section-item-paragraph">
      <b>When to use a patch changeset:</b>
    </p>
    <pre>
      <code>{`---
"@base-xui/react-xui": patch
---

Fixed Button focus state not showing in Firefox browsers`}</code>

</pre>

  </div>
</div>

<div className="sb-section">
  <div className="sb-section-item">
    <h3 className="sb-section-item-heading">MINOR (0.x.0)</h3>
    <p className="sb-section-item-paragraph">
      Minor versions add new functionality in a backward-compatible manner.
    </p>
    <p className="sb-section-item-paragraph">
      <b>Examples:</b>
    </p>
    <ul>
      <li>Adding new components</li>
      <li>Adding new props to existing components</li>
      <li>Adding new variants to existing components</li>
      <li>Adding new helper functions/utilities</li>
      <li>Expanding component functionality without breaking changes</li>
      <li>Deprecating features (but not removing them yet)</li>
    </ul>
    <p className="sb-section-item-paragraph">
      <b>When to use a minor changeset:</b>
    </p>
    <pre>
      <code>{`---
"@base-xui/react-xui": minor
---

Added new \`ghost\` variant to Button component`}</code>

</pre>

  </div>
</div>

<div className="sb-section">
  <div className="sb-section-item">
    <h3 className="sb-section-item-heading">MAJOR (x.0.0)</h3>
    <p className="sb-section-item-paragraph">
      Major versions contain <b>breaking changes</b> that require users to modify their existing code when upgrading.
    </p>
    <p className="sb-section-item-paragraph">
      <b>Examples:</b>
    </p>
    <ul>
      <li>Renaming component props or removing public props</li>
      <li>Changing the default behavior of a component</li>
      <li>Removing or renaming components/APIs</li>
      <li>Changing the return type/structure of component functions</li>
      <li>Major UI redesigns that affect component behavior</li>
      <li>Changes to the minimum supported React version</li>
    </ul>
    <p className="sb-section-item-paragraph">
      <b>When to use a major changeset:</b>
    </p>
    <pre>
      <code>{`---
"@base-xui/react-xui": major
---

Renamed the \`variant\` prop to \`appearance\` in Button component`}</code>

</pre>

  </div>
</div>

<div className="sb-section">
  <div className="sb-section-item">
    <h2 className="sb-section-item-heading">Decision Guidelines</h2>
    <p className="sb-section-item-paragraph">
      When deciding which version to increment, ask yourself:
    </p>
    <ol>
      <li>
        <b>Am I fixing a bug or making a small improvement?</b> → <b>PATCH</b>
      </li>
      <li>
        <b>Am I adding new features/functionality?</b> Is everything backward
        compatible? → <b>MINOR</b>
      </li>
      <li>
        <b>Is this a breaking change?</b> Will users need to update their code?
        → <b>MAJOR</b>
      </li>
    </ol>
  </div>
</div>

<div className="sb-section">
  <div className="sb-section-item">

        Adding a <code>size="xl"</code> option to buttons that previously only

      </li>
      <li>

      </li>

    <h3 className="sb-section-item-heading">Major Version Changes</h3>

        Changing <code>&lt;Button variant="primary" /&gt;</code> to require{" "}
        <code>&lt;Button appearance="primary" /&gt;</code>
      </li>
