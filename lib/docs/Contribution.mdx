    <p className="sb-section-item-paragraph">

        <b>main</b> – Production-ready code. Feature and bug fix branches are

      </li>

      </li>
      <li>
        <b>XX/feature/*</b> – New features or enhancements, prefixed with
        owner's initials (e.g., <code>JD/feature/button-component</code> for
        John Doe)
      </li>
      <li>
        <b>XX/fix/*</b> – Bug fixes, prefixed with owner's initials (e.g.,{" "}
        <code>JD/fix/button-focus-issue</code>)
      </li>
      <li>
        <b>XX/hotfix/*</b> – Emergency fixes for production, prefixed with
        owner's initials (e.g., <code>JD/hotfix/critical-bug</code>)
      </li>
    </ul>
  </div>
</div>

        <p>
          Please see

      </li>
      <li>
        <b>Commit your changes</b> (following conventional commits):
        <pre>
          <code>{`git add .

        </pre>
      </li>
      <li>
        <b>Push your branch and open a PR to development:</b>
        <pre>
          <code>
            git push -u origin XX/feature/your-feature-name # Replace XX with
          </code>

          <li>Ensure CI checks pass</li>

        </ul>
      </li>
    </ol>
  </div>

  <div className="sb-section-item">

    <ol>
      <li>
        <b>Create a PR from development to main</b>

      </li>
      <li>

        <ul>

          </li>
